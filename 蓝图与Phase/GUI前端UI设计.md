# RouteDev 前端 UI 设计文档

> 基于《GUI化计划表.md》的审美增强版
> 风格定位：像 Trae / Cursor / Linear 一样干净、现代、有秩序感的开发者工具
> 设计原则：深色为主、留白克制、信息分层清晰、动画细腻不打扰

---

## 一、设计语言 Design Language

### 1.1 设计关键词

- **秩序感**：网格对齐、固定节奏、层级明确
- **克制**：少而精的颜色、无多余装饰、内容优先
- **沉浸**：深色主界面，让代码和对话成为焦点
- **回应**：每个交互都有细腻的反馈（hover、focus、loading、success）

### 1.2 色彩系统 Color System

| Token | 色值 | 用途 |
|-------|------|------|
| `--bg-primary` | `#0B0B0B` | 主背景（比纯黑更柔和） |
| `--bg-secondary` | `#141414` | 侧边栏、面板背景 |
| `--bg-tertiary` | `#1B1B1B` | 卡片、输入框、hover 背景 |
| `--bg-hover` | `#242424` | hover 状态 |
| `--bg-active` | `#2A2A2A` | active / selected 状态 |
| `--border-default` | `#27272A` | 默认边框 |
| `--border-subtle` | `#18181B` | 弱分隔线 |
| `--border-focus` | `#6366F1` | focus 边框 |
| `--text-primary` | `#FAFAFA` | 主文本 |
| `--text-secondary` | `#A1A1AA` | 次要文本 |
| `--text-tertiary` | `#71717A` | 禁用 / hint |
| `--accent-primary` | `#6366F1` | 主品牌色 Indigo |
| `--accent-primary-hover` | `#4F46E5` | 主色 hover |
| `--accent-secondary` | `#22D3EE` | 辅助色 Cyan（状态、高亮） |
| `--accent-success` | `#34D399` | 成功 |
| `--accent-warning` | `#FBBF24` | 警告 |
| `--accent-error` | `#F87171` | 错误 |
| `--accent-magenta` | `#E879F9` | AI 思考中 / 特殊强调 |

**渐变：**
- AI 流式输出光晕：`linear-gradient(90deg, #6366F1 0%, #22D3EE 100%)`
- 状态栏 active 指示器：`radial-gradient(circle, #34D399 0%, #22C55E 70%, transparent 100%)`

### 1.3 字体系统 Typography

| 层级 | 字体 | 字号 | 字重 | 行高 | 字间距 |
|------|------|------|------|------|--------|
| 标题 H1 | Inter | 24px | 600 | 32px | -0.02em |

### 1.4 间距系统 Spacing

以 `4px` 为基准网格：

| Token | 值 | 用途 |
|-------|-----|------|
| `space-1` | 4px | 图标内边距、细缝 |
| `space-2` | 8px | 小组件间距 |
| `space-3` | 12px | 卡片内边距 |
| `space-4` | 16px | 面板内边距 |
| `space-5` | 20px | 页面级间距 |
| `space-6` | 24px | 大区块间距 |
| `space-8` | 32px | 页面两侧留白 |

### 1.5 圆角 Radius

| Token | 值 | 用途 |
|-------|-----|------|
| `radius-sm` | 4px | 标签、小按钮 |
| `radius-md` | 8px | 输入框、卡片 |
| `radius-lg` | 12px | 大卡片、弹窗 |
| `radius-xl` | 16px | 主窗口、侧边栏项目 |
| `radius-full` | 9999px | pill 按钮、头像 |

### 1.6 阴影 Shadow

| Token | 值 | 用途 |
|-------|-----|------|
| `shadow-sm` | `0 1px 2px rgba(0,0,0,0.3)` | 卡片hover |
| `shadow-md` | `0 4px 12px rgba(0,0,0,0.4)` | 下拉菜单、tooltip |
| `shadow-lg` | `0 12px 32px rgba(0,0,0,0.5)` | 弹窗、模态框 |
| `shadow-glow` | `0 0 24px rgba(99,102,241,0.15)` | AI 激活状态 |

### 1.7 动画 Animation

| 场景 | 时长 | 缓动 |
|------|------|------|
| hover 背景色 | 150ms | `cubic-bezier(0.4, 0, 0.2, 1)` |
| 消息出现 | 300ms | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 侧边栏切换 | 250ms | `cubic-bezier(0.4, 0, 0.2, 1)` |
| 弹窗出现 | 200ms | `cubic-bezier(0.16, 1, 0.3, 1)` |
| 流式输出光标闪烁 | 800ms | `steps(2, start)` |
| 工具卡片展开 | 250ms | `cubic-bezier(0.4, 0, 0.2, 1)` |

## 二、整体布局 Layout

### 2.1 窗口结构

采用 **"无标题栏 + 自定义标题栏"** 设计（Electron frameless window）：

```
┌──────────────────────────────────────────────────────────────────┐
│  ◉ RouteDev                              ─   □   ×   │ 标题栏 48px  │
├──────────┬───────────────────────────────────────────────────────┤
│          │                                                        │
│ Sidebar  │                    主内容区                              │
│  64px     │                                                        │
│          │   根据当前页面渲染 Chat / Settings / Token / Trace     │
│          │                                                        │
├──────────┴───────────────────────────────────────────────────────┤
│  Status Bar 36px                                                   │
└──────────────────────────────────────────────────────────────────┘
```

**标题栏 48px：**
- 左侧：品牌圆点 `#6366F1` + "RouteDev" 文字
- 右侧：窗口控制按钮（macOS 风格三个圆点，Windows 风格最小化/最大化/关闭）
- 中间：当前项目名称（如 `Agent`）

**侧边栏 64px：**
- 垂直图标导航，选中项有左侧 3px 高亮条 + 背景色变化
- 底部设置图标

**状态栏 36px：**
- 左侧：当前模型、tier、工作模式
- 中间：token 消耗 / 预算
- 右侧：连接状态、AI 状态指示器

### 2.2 页面切换

- 默认页面：Chat
- 切换动画：当前页淡出 + 新页淡入，200ms
- 记住上次退出时的页面

- 左侧：页面图标 + 标题 "当前对话"
- 右侧：
  - 「+ 新对话」按钮（secondary 样式）
  - 「编辑目标」按钮（只有在 goal 模式下显示）

### 3.3 消息气泡设计

#### User Message

```
┌──────────────────────────────────────────┐
│                                          │
│  帮我修复 tests/utils.test.ts 里的问题    │
│                                          │
│                                   12:34  │

- 靠右对齐
- 背景：`#1B1B1B`
- 最大宽度 80%
- 圆角：`radius-lg`，右下圆角略小（message tail 效果）
- 时间戳：`text-tertiary`，11px

#### AI Message

- 靠左对齐
- 背景：透明，左侧有 2px 渐变竖线 `#6366F1 → #22D3EE`
- 最大宽度 90%
- 包含：
  - 头部：RouteDev 小图标 + "RouteDev" 标签
  - 内容区：Markdown 渲染
  - 底部：操作栏（复制、重新生成、分支、展开思考过程）

### 3.4 流式输出效果

- AI 回复时，文字逐字出现
- 光标：底部有闪烁的 `▋`，颜色 `#22D3EE`
- 流式过程中，消息卡片有微弱的 `shadow-glow`
- 完成后光标消失，出现操作栏

### 3.5 工具调用卡片

```

│ }                                                            │

```

- 默认折叠，显示工具名 + 耗时 + 状态图标
- 展开后显示参数和结果
- 状态：
  - 运行中：旋转的 `#6366F1` 圆环
  - 成功：绿色对勾
  - 失败：红色感叹号
  - 等待确认：黄色问号

### 3.6 进度播报

```

当前步骤：运行测试验证修复

- 出现在 AI 消息内部的独立区块
- 进度条高度 4px，圆角 2px
- 已完成段 `#6366F1`，未完成段 `#27272A`
- 步骤切换时有宽度动画

### 3.7 输入框设计

```
┌─────────────────────────────────────────────────────────────────┐
│  ⌘  │  输入消息，使用 / 查看命令，拖拽文件到此处...            │  ↑  │

- 高度：最小 56px，最大 200px（自适应）
- 背景：`#141414`
- 边框：1px `#27272A`，focus 时变为 `#6366F1` + glow
- 左侧：语音输入按钮（可选）
- 右侧：
  - 「工具」按钮：快速选择常用工具
  - 「发送」按钮：圆形主色按钮，hover 放大 1.05 倍
- 拖拽文件到输入框时：边框变 `#22D3EE`，显示 "释放以上传文件"

### 3.8 命令补全面板

输入 `/` 后弹出：

```
┌─────────────────────┐
│ /build   执行模式    │
│ /plan    规划模式    │

- 浮层在输入框上方
- 选中项高亮 `#242424`
- 命令分类显示（不同颜色标签）

### 3.9 确认对话框

危险操作弹出：

```
┌─────────────────────────────────────────┐
│                                         │
│   ⚠️  确认执行以下操作？                  │
│                                         │
│   工具：shell_exec                       │
│   命令：npm test                         │
│                                         │
│        [取消]      [确认执行]            │
│                                         │
└─────────────────────────────────────────┘
```

- 模态框 + 背景遮罩 `rgba(0,0,0,0.6)`
- 红色强调危险操作
- 「确认执行」按钮有 300ms 延迟出现，防止误触

## 十二、实现建议

1. **先实现 Chat 页面**：这是核心使用场景，验证 IPC 通信后再扩展其他页面
2. **使用 Tailwind CSS + Shadcn/UI 组件**：快速实现设计语言一致性
3. **图标统一用 Lucide**：避免混用多个图标库造成风格不一致
4. **动画适度**：开发者工具不宜过度动画，保持 200-300ms 的细腻反馈即可
5. **可访问性**：确保所有交互可通过键盘完成，颜色对比度符合 WCAG AA
6. **预留扩展性**：Settings 左侧导航、Provider 卡片等设计成可动态添加项
