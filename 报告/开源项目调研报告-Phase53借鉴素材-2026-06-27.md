## 一、各项目调研摘要

### 1. microsoft/agent-governance-toolkit

**定位**：微软开源的 Agent 治理 SDK（MIT），为 AI Agent 提供确定性、应用层的策略执行中间件。核心理念是"Prompt 级别的安全指令不可靠，必须用确定性代码强制执行"。

**核心组件**：
- **PolicyEngine**：YAML/JSON 声明式策略规则，支持 deny-overrides / allow-overrides / priority-first-match / most-specific-wins 四种冲突解决策略，内置速率限制和审批工作流。Fail-closed 默认策略（无匹配规则时拒绝）。
- **AgentIdentity**：Ed25519 加密身份 + DID 格式，能力模型访问控制。
- **TrustManager**：贝叶斯信任评分（0-1），时间衰减，四级信任分层（Untrusted → Provisional → Trusted → Verified），磁盘持久化。
- **AuditLogger**：SHA-256 哈希链追加日志，timingSafeEqual 防篡改，溢出后保留接缝哈希。
- **RingEnforcer**：四环特权模型（Ring0 内核级 → Ring3 最受限），Agent 动作映射到所需环级。
- **KillSwitch**：紧急终止 + 补偿回调 + 替代 Agent 交接。
- **McpSecurityScanner**：扫描 MCP 工具定义的 4 类威胁——工具投毒（正则匹配注入模式）、名称仿冒（Levenshtein 距离 ≤2）、隐藏指令（零宽 Unicode + 同形字）、地毯式替换（长描述 + 指令模式）。
- **PromptDefenseEvaluator**：12 向量系统提示词安全评估（OWASP 对齐），A-F 评级。
- **CascadeContainment**：熔断器 + 依赖图分析 + 自动隔离 + 自动回滚。
- **ContextPoisoningDetector**：检测记忆/上下文中的提示注入（熵分析 + 模式匹配 + 重复检测）。

**多语言 SDK**：TypeScript / Python / .NET / Rust / Go，覆盖 20+ AI 框架集成。

### 2. Panniantong/Agent-Reach

**定位**：Python 实现的"能力层"（非封装层），为 AI 编码 Agent 提供跨 13+ 平台的互联网内容访问能力。核心洞察：Agent Reach 不读取内容本身，只负责安装、配置、健康检查和路由上游工具，Agent 直接调用上游 CLI。

**核心模式**：
- **Channel 策略模式**：`Channel` 抽象基类定义 `can_handle(url)` + `check()` + 有序后端列表。每个平台是具体子类。
- **有序后端路由**：每个频道声明优先级排序的后端列表，首个可用后端获胜。用户可通过配置强制指定后端。
- **`agent-reach doctor`**：一键健康检查，使用真实子进程探测（不是 `shutil.which()`），区分 missing / broken / timeout 三种失败模式。
- **Probe 层**：检测 Python 升级后 pipx/uv tool 安装的陈旧 venv 垫片（shim 在 PATH 上但解释器已消失）。
- **Cookie 自动提取**：通过 rookiepy 从 Chrome/Firefox/Edge 直接读取浏览器 Cookie。
- **环境自动检测**：评分式指标（SSH +2、Docker +2、无显示器 +1、云 VM UUID +2），阈值判定本地/服务器。
- **转录管线**：yt-dlp 下载 → ffmpeg 压缩 → Whisper API 转录，Provider 降级链（Groq → OpenAI）。

### 3. NVIDIA/SkillSpector

**定位**：NVIDIA 开源的 AI Agent 技能安全扫描器。研究数据：26.1% 的 Agent 技能包含漏洞，5.2% 显示恶意意图。

**核心架构**：
- **LangGraph DAG 工作流**：`StateGraph` 编译为确定性执行图，`Annotated[list[Finding], operator.add]` 归约器支持并行分析器并发追加发现。
- **双阶段分析**：Phase 1 静态（15 个正则/AST/YARA 分析器并行），Phase 2 语义（3 个 LLM 分析器 + 元分析器过滤误报）。
- **元分析器的严重性地板**：CRITICAL 和 HIGH 静态发现**即使 LLM 拒绝也保留**（标记 `llm-unconfirmed`），防止对抗性技能内容欺骗 LLM 隐藏严重问题。
- **风险评分算法**：CRITICAL=50, HIGH=25, MEDIUM=10, LOW=5 基础分，**同规则递减**（第 2 次 0.5x，第 3 次 0.25x，之后忽略），置信度缩放，可执行文件 1.3x 乘数，封顶 0-100。
- **Fail-closed 设计**：LLM 完全失败时，所有发现保留并显示（多显示比静默丢弃更安全）。
- **基线抑制系统**：Glob 规则（人类编写，漂移容忍）+ SHA-256 指纹（机器生成，精确匹配）双重误报管理。
- **MCP 服务器模式**：暴露 `scan_skill` 工具，任何 MCP 兼容的 Agent 可在安装技能前调用扫描。
- **诚实 LLM 记账**：输出始终报告 `llm_requested` / `llm_available` / `llm_used` / `scan_mode`，不误导用户。
- **Token 预算感知分块**：大文件按模型上下文窗口安全分块，带重叠行上下文，asyncio.gather + 信号量隔离失败。

### 4. rohitg00/ai-engineering-from-scratch

**定位**：320+ 小时、435 课的 AI 工程课程，"从零构建"方法论。虽然不是单一应用，但编码了权威的 Agent 系统架构模式。

**核心可借鉴模式**：
- **ReAct 循环五必备组件**：交互历史、可执行函数目录、终止触发器、迭代上限、输出格式化器。
- **三层虚拟记忆模型**（MemGPT/Letta）：主上下文（RAM，固定大小）→ 外部上下文（磁盘，向量+图谱+KV）→ 转移函数（页错误）。
- **四种编排拓扑**：Supervisor-Worker（最有组织）、Swarm（最快）、Hierarchical（记忆限制时使用）、Debate（最贵但最准）。渐进采用规则：单 Agent → Supervisor → Swarm → Hierarchy。
- **Plan-and-Execute (ReWOO)**：Planner 生成 DAG → Worker 拓扑排序并行执行 → Solver 综合。比 ReAct 节省 5x Token。
- **五种工作流模式**：提示链、路由、并行化、编排-工人、评估-优化。
- **持久执行四要素**：确定性编排逻辑、不可预测任务（带日志）、持久事件记录、重放恢复机制。
- **多会话交接**：在 50-75% 上下文预算时执行交接（不是 95%），交接文档必须包含下一个任务。
- **信任边界崩塌**：外部数据（工具结果、网页内容）注入恶意指令，模型将其视为权威来源——Agent 系统的 #1 安全风险。
- **技能库模式（Voyager）**：技能存储为可运行脚本（非仅提示），通过"查询到描述相似度"检索，版本化、可组合。
- **计算机使用安全**：所有环境数据（截图、DOM、文件）是不可信的，只有直接用户指令算作权限授权。

### 5. addyosmani/agent-skills

**定位**：Google Chrome 团队工程负责人 Addy Osmani 的 24 个生产级工程工作流技能，沿软件开发生命周期组织。

**核心设计**：
- **生命周期对齐**：8 个斜杠命令（`/spec` → `/plan` → `/build` → `/test` → `/review` → `/webperf` → `/code-simplify` → `/ship`）覆盖完整开发周期。
- **元技能路由**：`using-agent-skills` 元技能分析任务并自动路由到正确工作流。
- **反合理化表**：每个技能包含预置反驳表，系统性对抗 AI 常见借口（如"我以后再加测试"被逐条反驳）。
- **证据驱动验证**：技能要求具体证据而非断言——"Seems right"被明确拒绝作为证据。
- **对抗性审稿人**：4 个专家审稿人格（adversarial reviewer personas），从不同视角挑战 Agent 输出。
- **共享参考清单**：跨领域关注点（安全、无障碍、性能）抽取为共享参考文件，多个技能复用。

### 6. anthropics/skills

**定位**：Anthropic 官方 Agent Skills 框架——模块化、动态加载的能力包规范。包含开源示例技能和 Claude 原生文档能力（docx, pdf, pptx, xlsx）的生产技能。

**核心设计**：
- **正式规范**（agentskills.io/specification）：严格的 SKILL.md schema，name（≤64 字符）和 description（≤1024 字符）必填，description 驱动发现。
- **渐进/分层加载**：Tier 1 仅 name + description（用于路由发现）→ Tier 2 完整 SKILL.md（技能激活时）→ Tier 3 补充文件按需加载。
- **文件夹自包含结构**：`SKILL.md` + `scripts/` + `references/` + `static/`。
- **意图/知识/执行分离**：SKILL.md 处理路由和指令，references/ 提供领域知识，scripts/ 处理确定性执行。
- **Eval 驱动开发**：skill-creator 元技能要求写测试提示 → 并行评估 → 评分 → 迭代至通过。
- **插件市场集成**：注册仓库为市场源，CLI/UI 安装。

### 7. LMCache/LMCache

**定位**：LLM 推理的 KV 缓存管理层，将瞬态 KV 状态转化为持久、可复用的"一等数据对象"。

**核心模式**：
- **内容可寻址分块缓存**：输入 Token 按固定大小分块（默认 256 Token），每块 SHA-256 链式哈希（与前缀哈希链接），相同 Token 序列自动产生相同缓存键。
- **分层存储**：GPU VRAM → CPU RAM → 本地 SSD → 远程服务器（Redis, S3）。
- **Skip-existing 优化**：存储时逐块顺序检查，遇到第一个未缓存的块即停止，避免共享前缀的冗余写入。
- **写穿/读穿混合后端**：写入同时到本地和远程；读取先查本地，未命中则查远程并提升（promote）。
- **管线化检索**：并行网络获取和反序列化在不同线程中执行。
- **非前缀 KV 复用（CacheBlend）**：允许缓存块在提示中任意位置使用（非仅前缀），选择性重算受影响的注意力 Token。
- **注意**：LRU 驱逐策略在代码中标记为 TODO 尚未实现。

### 8. jnMetaCode/agency-agents-zh

**定位**：266 个即插即用 AI 专家人格定义（中文社区版），覆盖 20 个企业部门。配套 agency-orchestrator 提供多 Agent DAG 编排。

**核心模式**：
- **Agent-as-Markdown Schema**：每个 Agent 是结构化 Markdown（YAML frontmatter + Identity/Mission/Rules/Workflow/Deliverables/Style/Metrics 段落），用户可编辑、版本可控、LLM 无关。
- **DAG 工作流引擎**：解析 `depends_on` 数组，独立节点并行执行，`{{variable}}` 模板标签传递数据。
- **可复用团队**：将专家组合保存为 `.team.yaml` 文件，跨任务复用。
- **三种重试 + 升级**：有界重试循环 + 人类升级，防止失控成本和无限循环。
- **熔断器模式**：CLOSED/OPEN/HALF-OPEN 状态，Agent 调用连续失败时优雅降级。
- **跳间上下文压缩**：摘要或结构化状态在 Agent 跳间传递，防止 Token 膨胀。
- **Electron 桌面应用**：编排器将 Node.js 和所有依赖打包在 Electron 内，消除"先装 Node"的摩擦。
- **成功指标量化**：每个 Agent 定义可量化 KPI（如"Lighthouse > 90"、"LCP < 2.5s"），系统可编程验证输出。

### 9. affaan-m/ECC (Everything Claude Code)

**定位**：v2.0.0，271 个可复用技能 + 20+ 专家 Agent + 钩子自动化系统，作为 AI 编码助手的"操作系统层"。

**核心模式**：
- **271 个技能目录**：按领域组织（前端/后端/语言/DevOps/安全/测试），Markdown + YAML frontmatter。
- **声明式钩子系统**：`hooks.json` 定义事件（PreToolUse / PostToolUse / Stop / SessionStart 等）+ 匹配器（正则工具名）+ 处理器（Node.js 脚本）。支持 async、timeout、分层激活（minimal/standard/strict）。
- **GateGuard（事实强制）**：阻止首次 Edit/Write 任何文件，要求 Agent 先调查文件的导入者、数据模式和用户指令。
- **配置保护守卫**：阻止 AI 弱化自己的约束（如禁用 linting 规则来消除错误）。
- **上下文预算监控**：Token 耗尽、成本超支、范围蔓延、工具循环时注入警告。
- **累加器 + 批处理**：Post-edit 钩子累积编辑路径，Stop 钩子统一批量格式化 + 类型检查。
- **多分层操作模式**：minimal（最少钩子）/ standard / strict（全治理），用户选择自动化激进度。
- **Markdown-as-配置**：技能、Agent、规则均为 `.md` 文件，开发者友好、版本控制友好。
- **ECC2 (Rust α)**：Rust 控制面板 + 终端 UI + SQLite 会话存储 + 后台守护进程。

## 二、跨项目共性模式与 RouteDev 借鉴矩阵

| **健康检查/Doctor** | Agent-Reach | 无 | 中：一键诊断所有集成状态 |
| **有序后端路由** | Agent-Reach | 路由是单模型选择 | 低：现有路由已足够 |

## 三、建议纳入 Phase 53 的落地任务

### 第一梯队：高优先级（安全 + 核心架构）

**T1. Agent 治理管线接入（借鉴 AGT）**
将 app-init.ts 中已创建但未连接的策略引擎（PolicyEngine, IntentGuard, Arbitration）正式接入 ReActAgentLoop。添加 setter 或中间件注册点，实现每个工具调用前的策略检查。包括：
- 为 ReActAgentLoop 添加 `setPolicyEngine()` setter
- 在中间件管线的 `onActing` 阶段插入策略检查
- 配置 fail-closed 默认策略
- ≥3 测试

**T2. 哈希链审计日志（借鉴 AGT）**
升级现有 AuditLogger，实现 SHA-256 链式哈希追加日志：
- 每条记录的 hash = SHA-256(timestamp + agentId + action + decision + previousHash + metadata)
- 创世哈希为 64 个零
- 溢出时保留接缝哈希
- timingSafeEqual 防篡改验证
- 导出为 JSON 供用户审查
- ≥2 测试

**T3. MCP 安全扫描器（借鉴 AGT + SkillSpector）**
为 MCP 工具定义添加安全扫描：
- 工具投毒检测（正则匹配 `<system>`, `ignore previous` 等注入模式）
- 名称仿冒检测（与已知工具名的 Levenshtein 距离 ≤2）
- 隐藏指令检测（零宽 Unicode + 同形字）
- 地毯式替换检测（描述 >500 字符 + 2+ 指令模式）
- 扫描结果在工具注册时展示风险等级
- ≥3 测试

**T4. 技能安全扫描门控（借鉴 SkillSpector）**
第三方技能安装前的安全门控：
- 静态分析：正则匹配 17 类漏洞模式
- 风险评分：0-100 分，同规则递减算法
- 安装拦截：score > 50 的技能拒绝自动安装，需用户确认
- 基线抑制：Glob + SHA-256 指纹双重误报管理
- 诚实报告：标注扫描模式（static-only vs static+llm）
- ≥3 测试

**T5. 前缀感知上下文缓存（借鉴 LMCache）**
实现内容可寻址的上下文缓存，节省 API Token：
- 对话历史按固定 Token 窗口分块（256 或 512 Token）
- 每块 SHA-256 链式哈希（与前缀哈希链接）
- L1 内存（LRU Cache）+ L2 IndexedDB 分层存储
- Skip-existing 写入优化
- 读穿（Read-through）提升机制
- 与 Anthropic prompt caching API 对齐（如果可用）
- ≥3 测试

**T6. 配置保护守卫（借鉴 ECC）**
防止 Agent 弱化自身约束：
- 拦截对 routedev 配置文件（.routedev.yaml, config.yaml）的修改
- 拦截对权限规则、安全策略、钩子配置的修改
- 区分"修复代码"和"弱化配置"两种意图
- 首次触发时警告用户并记录审计日志
- ≥2 测试

### 第二梯队：中优先级（功能增强）

**T7. 渐进式技能加载（借鉴 anthropics/skills）**
实现三级技能加载，节省上下文窗口：
- Tier 1：启动时仅加载所有技能的 name + description（用于路由匹配）
- Tier 2：技能激活时加载完整 SKILL.md（≤500 行）
- Tier 3：按需加载 references/ 和 scripts/
- 修改 SkillLifecycle 和 SkillFlowEngine 的加载逻辑
- ≥2 测试

**T8. 上下文预算监控与告警（借鉴 ECC + ai-eng）**
在 Agent 运行时添加实时预算监控：

**T10. 熔断器模式（借鉴 agency-agents + AGT）**

- 超时后 → HALF-OPEN，允许单次试探
- 试探成功 → CLOSED，失败 → 回到 OPEN
- 与 SubAgentScoreCard 和 WorkerExecutor 集成

## 五、总结

9 个开源项目提供了丰富的架构借鉴素材。**最高价值的借鉴方向**是：

1. **安全治理**（AGT + SkillSpector）：策略引擎接入、哈希链审计、MCP 安全扫描、技能安全门控——RouteDev 已有策略引擎的骨架但未接入，这是最紧迫的修复项。

2. **上下文效率**（LMCache + anthropics/skills）：前缀感知缓存 + 渐进式技能加载——直接影响 Token 成本和上下文窗口利用率。

3. **运行时韧性**（agency-agents + ECC）：DAG 工作流 + 熔断器 + 配置保护 + 上下文预算监控——提升 Agent 长时间运行的可靠性。

4. **代码卫生**（基于审查报告）：22 个死代码文件的清理 + "装配但未连接"模块的验证——这是 Phase 53 的前置工作，确保新功能不会建立在死代码之上。
